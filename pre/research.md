from genlayer import *
import json
import typing

class GnothiNewsOracle(gl.Contract):
    market_resolved: bool
    market_result: str
    target_question: str
    evidence_url: str

    def __init__(self, question: str, url: str):
        self.market_resolved = False
        self.target_question = question
        self.evidence_url = url
        self.market_result = "PENDING"

    @gl.public.write
    def resolve_market(self) -> typing.Any:
        if self.market_resolved:
            return "Market is already resolved."

        def ai_consensus_block() -> str:
            # İnternet verisine erişim (gl.nondet.web.render veya get_webpage)
            web_data = gl.nondet.web.render(self.evidence_url, mode="text")
            
            # LLM Ajanlarına verilecek yapılandırılmış prompt
            prompt = f"""
            Aşağıdaki web sitesi metnini dikkatlice inceleyerek soruyu yanıtla.
            Soru: {self.target_question}
            Kanıt Metni: {web_data}
            
            Eğer haber metni sorunun gerçekleştiğini açıkça kanıtlıyorsa 'SIDE_A', gerçekleşmediğini veya aksi bir durum olduğunu kanıtlıyorsa 'SIDE_B', 
            olay henüz gerçekleşmemişse veya kanıt yetersizse 'UNDECIDED' sonucuna var.
            Sadece JSON formatında yanıt ver. Örnek: {{"decision": "SIDE_A"}}
            """
            
            # LLM'in promptu işlemesi
            result = gl.exec_prompt(prompt).replace("```json", "").replace("```", "").strip()
            return result

        # Konsensüs mekanizmasının tetiklenmesi
        final_result = gl.eq_principle.strict_eq(ai_consensus_block)
        
        try:
            result_json = json.loads(final_result)
            decision = result_json.get("decision", "UNDECIDED")
            
            if decision in:
                self.market_resolved = True
                self.market_result = decision
            
            return decision
        except Exception as e:
            raise gl.UserError("JSON parse error from LLM output")

Gnothi için bu, JSON içeriğinin anlamsal olarak aynı karar parametresine (SIDE_A, SIDE_B veya UNDECIDED) sahip olup olmadığını kontrol etmek anlamına gelir:Python    def is_acceptable(self, validator_output: str, leader_output: str) -> bool:
        """
        Liderin çıktısı ile doğrulayıcının çıktısının anlamsal olarak
        aynı kararı taşıyıp taşımadığını kontrol eder.
        """
        def extract_decision(text: str) -> str:
            # JSON formatını parse et ve karar parametresini güvenli bir şekilde çek
            try:
                # LLM'lerin ekleyebileceği markdown kod bloklarını temizle
                clean_text = text.replace("```json", "").replace("```", "").strip()
                data = json.loads(clean_text)
                return data.get("decision", "UNKNOWN").upper()
            except:
                return "ERROR"
                
        val_decision = extract_decision(validator_output)
        lead_decision = extract_decision(leader_output)
        
        # Eğer ikisi de anlamlı bir karar üretmişse ve bu kararlar eşleşiyorsa kabul et
        if lead_decision in:
            return val_decision == lead_decision
            
        return False

Bu tür tahmin piyasası kitleri, likiditeyi yönetmek ve kullanıcıların bahislerini anlık fiyatlara dönüştürmek için Robin Hanson tarafından icat edilen Logaritmik Piyasa Puanlama Kuralı (LMSR - Logarithmic Market Scoring Rule) tabanlı Otomatik Piyasa Yapıcı (AMM) algoritmalarını kullanır. Gnothi'nin Aşama 2 (Ticaret) evresi tamamen bu matematiğe dayanmaktadır.LMSR algoritması, piyasada işlem gören her bir olasılık durumu için toplam likidite maliyetini merkezi bir maliyet fonksiyonu üzerinden belirler. $n$ adet olasılık durumunun (örneğin EVET ve HAYIR olmak üzere 2 durum) bulunduğu bir piyasada, her bir olasılık için o ana kadar satılmış olan hisse miktarlarını ifade eden vektör $q = (q_1, q_2,..., q_n)$ olsun. LMSR maliyet fonksiyonu şu şekilde tanımlanır :$$C(q) = b \ln \left( \sum_{j=1}^{n} e^{q_j / b} \right)$$Burada $b$, piyasanın "likidite parametresi" (Market funding parameter) olarak adlandırılır. $b$ değeri ne kadar büyükse, piyasada büyük hacimli işlemler yapıldığında fiyat kayması (slippage) o kadar az olur. Piyasayı başlatan kişinin (Creator) yatırması gereken maksimum teminat miktarı $F = b \ln(n)$ formülü ile belirlenir.Bir tüccar (kullanıcı veya bot) sisteme gelip $i$ numaralı durumun gerçekleşme olasılığı üzerinden işlem yapmak istediğinde, hissenin anlık fiyatı ($p_i$), maliyet fonksiyonunun ilgili miktar üzerinden kısmi türevi alınarak hesaplanır :$$p_i(q) = \frac{\partial C}{\partial q_i} = \frac{e^{q_i / b}}{\sum_{j=1}^{n} e^{q_j / b}}$$Bu formülasyonun en büyüleyici özelliği, tüm $p_i(q)$ fiyatlarının her zaman 0 ile 1 arasında bir değer alması ve piyasadaki tüm olasılıkların fiyatları toplamının mutlak surette 1'e eşit ($\sum_{i} p_i(q) = 1$) olmasıdır. Bu sayede, fiyat doğrudan o olayın gerçekleşme "olasılığına" dönüşür.Kullanıcı bir tahmin yapıp $\Delta q_k$ miktarında hisse satın almak istediğinde, ödemesi gereken net tutar, maliyet fonksiyonunun yeni durumu ile eski durumu arasındaki farktan hesaplanır:$$\text{TradeCost} = C(q + \Delta e_k) - C(q)$$
Bu kusursuz matematik, Gnothi piyasalarının likidite sıkıntısı çekmeden, her zaman bir karşı taraf (counterparty) bulunmasına gerek kalmaksızın anında işlem görmesini sağlar.Kesin Uygun Puanlama Kuralları (SCEM) ve Teşvik Oyun TeorisiTahmin piyasalarının nihai amacı, toplumun (kitlelerin) dağınık bilgisini tek ve doğru bir olasılık sinyaline dönüştürmektir. Ancak bunu başarmak için, katılımcıları gerçek inançlarını bildirmeye (truth-telling) teşvik eden bir ödeme sistemine ihtiyaç vardır. Gnothi sisteminin Aşama 5'inde belirtilen SCEM (Strictly Proper Scoring Rules / Kesin Uygun Puanlama Kuralları) mekanizması, geleneksel bahis sitelerindeki "kazanan her şeyi alır" (winner-takes-all) mantığını yok ederek piyasayı bir bilimsel modelleme alanına dönüştürür.Bir puanlama kuralı $S(i, q)$, gerçekleşen nihai mutlak gerçek $i$'ye karşı kullanıcının bildirdiği $q$ tahminini skorlar. Eğer bir yatırımcının beklenen ödülünü maksimize etmesinin tek yolu, kendi içsel olarak inandığı gerçek olasılığı ($p$) sisteme girmesi ise, bu kural "Kesin Uygun" (Strictly Proper) olarak kabul edilir. SCEM mimarisi; karmaşık hidrolojik tahminlerde, enerji talep tahminlerinde ve otonom sistemlerin siber kredi değerlendirmelerinde bileşenlerin dürüstlüğünü sağlamak için literatürde başarıyla kullanılmış devasa bir teorik çerçevedir.Gnothi projesi, LMSR ile SCEM arasındaki matematiksel köprüyü Çapraz-Entropi Piyasa Puanlama Kuralı (Cross-Entropy Market Scoring Rule - CE-MSR) aracılığıyla kuracaktır. Kullanıcının sisteme $t$ anında girip, piyasa tahminini $q^{(t-1)}$'den $q^{(t)}$'ye güncellemesi durumunda alacağı net ödül (veya ceza) şu formülle hesaplanır:$$SCEM(r, q^{(t)}, q^{(t-1)}) = -H(r, q^{(t)}) + H(r, q^{(t-1)}) = \sum_{i} r_i \ln(q_i^{(t)}) - \sum_{i} r_i \ln(q_i^{(t-1)})$$Burada:$r$: GenLayer ağındaki yapay zeka ajanlarının belirlediği "Mutlak Gerçek" dağılımıdır (Örneğin SIDE_A gerçekleştiyse $r = $).$q^{(t-1)}$: Kullanıcı işlem yapmadan önceki piyasa olasılığıdır.$q^{(t)}$: Kullanıcının işlemi sonrası piyasanın geldiği yeni olasılık durumudur.Bu denklem, Gnothi'nin devrimci yanını ortaya koymaktadır: Eğer piyasa %30 oranında SIDE_A'nın gerçekleşeceğini düşünürken, bir analist (veya yapay zeka botu) gelip haklı bir kanıtla bunu %70'e çekerse ve GenLayer olay sonunda SIDE_A'yı doğru kabul ederse, bu analist devasa bir asimetrik ödül alır. Ancak piyasa zaten %95 oranında eminken %96'ya çeken kişi çok az bir ödül alır. Kullanıcı, yapay zekanın (GenLayer'ın) nihai olarak ne karar vereceğini en iyi tahmin etmeye çalışır. Bu yapı, yatırımcıların, olayın nihai sonucunu belirleyecek olan LLM ajanlarının referans beklentisini (reference agent's expected prediction) modellemesine olanak tanır. SCEM formülü, yanılanların teminatlarını matematiksel bir oranda keserken, sisteme erkenden doğru bilgi sağlayanları ödüllendirerek kitlelerin bilgeliğini kusursuzlaştırır.Bradbury Testnet Hackathon'u İçin Geliştirme Haritası ve pm-kit ModifikasyonlarıGenLayer'ın "Bradbury Builders Hackathon"u, yapay zekanın blokzincir konsensüsüne doğrudan katıldığı Bradbury Testnet üzerinde gerçekleştirilmektedir.
